*Documento rector de arte, UI, paleta, componentes y reglas visuales.*

  **Paquete documental para construcción web en Claude**

  -----------------------------------------------------------------------

# 1. Principio rector del diseño

-   La sensación que debe transmitir la web es calma operativa.

-   No debe sentirse como empresa de transporte tradicional, startup
    gritona ni dashboard frío y genérico.

-   Debe transmitir claridad, orden, foco, estabilidad, confianza
    operativa y simpleza inteligente.

# 2. Territorio visual

-   Concepto visual madre: Software que ordena el día.

-   Referencias correctas: paneles limpios, flujos claros, estados
    visibles, checklists, tarjetas estructuradas, mapas discretos,
    evidencia organizada.

-   Referencias incorrectas: motores, ruedas, carreteras eternas, humo
    industrial, containers y estética de taller.

# 3. Dirección de arte

-   Estilo recomendado: minimalismo operativo B2B.

-   Mucha respiración, grillas limpias, jerarquía visual fuerte, íconos
    sobrios, uso táctico del color, interfaces como parte del lenguaje
    gráfico y movimiento sutil.

# 4. Ruta visual elegida

-   Ruta A --- Sobria operativa (recomendada).

-   Fondos claros, bloques blancos o gris suave, un color profundo
    principal, un acento controlado, mockups elegantes y tipografía
    limpia.

-   Proporción recomendada de uso: 70% neutros, 20% primario, 10%
    acento.

-   El acento no debe gritar. El color debe ayudar a entender, no
    decorar.

# 6. Tipografía

-   Tipografía principal recomendada: Inter.

-   Alternativas válidas: Manrope, Geist, Plus Jakarta Sans.

-   Jerarquía sugerida: H1 48--64 px, H2 32--40 px, H3 24--28 px, cuerpo
    16--18 px.

# 7. Layout y composición

-   Ancho máximo de contenido: 1200--1280 px.

-   Secciones con padding vertical generoso.

-   Uso consistente de dos columnas para explicación + mockup.

-   Una idea fuerte por bloque; alternar texto + visual; lectura en
    zigzag simple.

# 9. Iconografía y visuales

-   Íconos lineales, simples y modernos.

-   Librerías sugeridas: Lucide, Tabler Icons o Phosphor en variante
    simple.

-   La UI del producto debe ser parte importante del lenguaje visual.

# 10. Mockups, fotos y recursos

-   Proporción recomendada: 60% UI/mockups, 20% diagramas o flujos, 20%
    fotografía contextual.

-   Mostrar tablero del día, pedidos, asignación, mapa simple, evidencia
    y cierre diario.

-   Evitar fotos stock genéricas de ejecutivos sonrientes o flotas
    gigantes irreales.

# 11. Motion

-   La animación debe servir a la comprensión, no al espectáculo.

-   Permitido: aparición suave, hover sutil, transición limpia de cards
    y menú mobile.

-   No permitido: scroll teatral, parallax pesado, loaders teatrales.

# 12. Do / Don't visual para Kiro

-   DO: usar aire, mostrar producto, priorizar claridad, trabajar con
    grilla, mantener coherencia entre secciones.

-   DON'T: abusar de gradientes, meter demasiadas fotos genéricas, usar
    estética de transporte pesado, parecer fintech o startup IA
    genérica.

# 13. Veredicto final

LIVIO debe verse como un software que trae calma y control a una
operación que antes vivía en desorden.
