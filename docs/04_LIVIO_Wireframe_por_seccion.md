# 1. Regla narrativa del Home

La home debe seguir esta progresión: entiendo qué es → me identifico con
el problema → veo cómo funciona → entiendo el valor → confirmo que calzo
→ confío → contacto.

# 2. Estructura general del Home

1.  Navbar

2.  Hero

7.  Alcance del MVP / qué incluye

8.  Video

9.  Diferenciación

10. Nosotros / confianza

11. CTA final + formulario

12. Footer

# 3. Wireframe por sección

# 4. Páginas secundarias

-   Cómo funciona: hero corto + flujo completo + mockups por etapa +
    bloque de cierre + CTA.

-   Soluciones: hero corto + grid de casos de uso + problemas comunes +
    diferencias vs operación manual + CTA.

-   Nosotros: hero corto + qué creemos + principios + manifiesto corto +
    CTA.

-   Contacto: hero corto + formulario principal + datos de contacto +
    expectativa de respuesta.

# 5. Reglas responsive

-   Mobile: todo esencial debe leerse sin esfuerzo; cards en stack; hero
    en bloque vertical; mockups simplificados.

-   Desktop: aprovechar aire, jerarquía y protagonismo de mockups.

-   No debe existir una sección que dependa de tener demasiado ancho
    para funcionar.

# 6. Qué no debe hacer Kiro

-   Inventar secciones extra sin sentido.

-   Llenar la home de texto largo.

-   Meter sliders innecesarios.

-   Poner múltiples CTAs compitiendo.

-   Convertir la web en un dashboard.

-   Usar estética de camión o transporte pesado.

# 7. Veredicto final

LIVIO no se presenta como una página bonita de software. Se presenta
como una máquina de claridad operativa.
