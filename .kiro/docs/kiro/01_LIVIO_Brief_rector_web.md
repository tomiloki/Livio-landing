# 1. Objetivo de la web {#objetivo-de-la-web}

- La web de LIVIO no debe sentirse como una landing genérica de software logístico ni como una página de optimización de rutas.

- Debe vender una idea más defendible: LIVIO ordena la operación diaria de distribución para que una pyme pueda operar con control, evidencia y cierre diario, sin depender del héroe de siempre.

# 2. Qué debe lograr la web {#qué-debe-lograr-la-web}

- Explicar en segundos qué hace LIVIO.

- Hacer que el cliente correcto se identifique rápido.

- Bajar la fricción y generar confianza B2B.

- Diferenciar a LIVIO de los ruteadores genéricos.

- Convertir visitas en reuniones o leads calificados.

# 3. Posicionamiento correcto {#posicionamiento-correcto}

- No venderse como app de camiones.

- No venderse como IA mágica que ahorra plata.

- No parecer un TMS enterprise complejo.

- Sí venderse como sistema operativo para la logística diaria.

- Sí vender orden, control, evidencia y cierre diario auditable.

# 4. Idea fuerza {#idea-fuerza}

- Frase madre recomendada: LIVIO ordena tu operación logística diaria para que puedas delegar, controlar y cerrar cada día con evidencia.

- Idea fuerza: No vendemos rutas. Vendemos operación bajo control.

# 5. Público objetivo {#público-objetivo}

- ICP primario: pymes y medianas con 5 a 20 vehículos, operación subdigitalizada, pedidos por WhatsApp / llamadas / Excel, planificación manual, múltiples paradas y alta dependencia del dueño o despachador.

- Rubros clave: agua embotellada, gas a domicilio, mayoristas con despacho propio, Horeca y distribución recurrente.

- Público secundario: dueños, operadores, jefes de operaciones y administradores.

# 6. Tono de marca {#tono-de-marca}

- Simple, clara, confiable, sobria, ordenada y cercana B2B.

- Hablar con autoridad operativa, no con humo startup.

- Evitar exceso de jerga tech, promesas infladas y tono futurista.

# 7. Concepto creativo {#concepto-creativo}

- Concepto central: Operación en orden. Crecimiento con control.

- Territorios válidos: del caos a la rutina, del héroe al sistema, del desorden a la trazabilidad, de apagar incendios a operar con método.

# 10. Principios de contenido {#principios-de-contenido}

- La web debe hablar más de operación que de rutas.

- Debe vender control y cierre diario antes que ahorro prometido.

- Debe hablarle a pymes reales, no a enterprise gigante.

- Debe ser sobria y clara, no espectacular por espectáculo.

- Debe dejar espacio para crecer a contenido, casos y SEO.

# 11. SEO y CRO a nivel rector {#seo-y-cro-a-nivel-rector}

- SEO v1: enfocarse en keywords con intención clara como software logístico para pymes, software de despacho, control de entregas, prueba de entrega y cierre operativo.

- CRO: CTA principal único Agendar demo; CTA secundario Evaluar mi operación; formulario al final y WhatsApp visible pero no invasivo.

# 12. Veredicto final {#veredicto-final}

- LIVIO debe presentarse como el sistema que convierte una operación logística desordenada en una rutina clara, delegable y controlada.

- No debe parecer una empresa de transporte ni un software más.
