Analiza el componente o sección que estoy referenciando y mejora su diseño visual.

Sigue este proceso:

1. **Auditoría visual**: Identifica problemas de spacing, jerarquía tipográfica, contraste, alineación y consistencia con el resto del sitio.

2. **Plan de mejoras**: Antes de tocar código, lista los cambios concretos que harás y por qué. Espera mi aprobación.

3. **Implementación**: Aplica los cambios usando Tailwind CSS. Si necesitas animaciones, usa CSS transitions o Framer Motion si ya está instalado.

Reglas:
- Mobile-first siempre. Verifica que se vea bien en 375px, 768px y 1280px.
- No cambies el copy/texto a menos que te lo pida explícitamente.
- Los mockups de UI dentro de la landing deben verse como software real, no como wireframes.
- Mantén coherencia con las secciones que ya están bien.
- Si detectas un problema de accesibilidad (contraste, alt tags, focus states), menciónalo.

Contexto: Este es el sitio público de Livio (liviogistics.com), una plataforma logística para pymes chilenas. El objetivo es generar confianza y captar empresas piloto.
